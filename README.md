# Trying to install OpenMoji-color-glyf_colr_0.ttf directly 🤢
This issue usually happens because **removing** a glyph from a font file (like the space character `U+0020`) often leaves behind a "ghost" entry with a default **advance width**. 

When a font has a space glyph with no outlines but a large width, the system still uses that width for every space you type, leading to those huge gaps between words.

If you used code (likely Python with `fontTools`) generated by Claude to edit your OpenMoji font, here is how to fix the "too many spaces" problem:

### 1. The Problem: "Empty" vs. "Removed"
If the script Claude gave you just cleared the glyph's contents but kept the mapping, the font still "claims" to have a space character. Because it's an emoji font, its default glyph size (the EM square) is often much larger than a standard text font (e.g., 1024 or 2048 units wide vs. 200–300 for a normal space).

### 2. The Solution: Proper Subsetting
Instead of just deleting the glyph data, you need to **completely remove the mapping** from the `cmap` table so the system falls back to your regular system font for spaces and digits.

If you are using a Python script, tell Claude to:
> "Modify the script to completely remove the entries for U+0020 (space) and the Basic Latin digits from the **cmap table**, and ensure the **hmtx table** does not contain entries for these characters."

### 3. Quick Fix Script (using FontTools)
If you want to try a manual fix, you can run this Python command (if you have `fonttools` installed) to "subset" the font and strip everything except what you need:

```bash
pip install fonttools
pyftsubset OpenMoji-color-glyf_colr_0.ttf --unicodes="U+1F000-1F9FF,U+2600-26FF" --layout-features="*"
```
*(Note: Adjust the unicode range to include the specific emojis you want to keep. This ignores the space/latin characters entirely.)*

### 4. Why it looks weird in the screenshot
In your screenshot, the message text itself has triple spaces: `latin   suplement`. This suggests that whatever tool was used to generate that instruction text (or the file caption) might have also been inserting extra characters or using a "Full-Width" space character common in some LLM formatting errors.

**To fix the font you've already made:**
Open it in a tool like **FontForge**, find the "Space" glyph (Unicode 32), and check its "Width." If it's something huge like 1000, change it to around **250** or **delete the glyph entirely** so your system uses the default font for spacing.
